Técnicas de Programação - Primeiro Bimestre

Em nosso projeto utilizaremos um banco de dados relacional para armazenar os dados, pois provê acesso facilitado aos mesmos e nos possibilita uma grande variedade de abordagens no tratamento das informações.

O modelo relacional não tem caminhos pré-definidos para se fazer acesso aos dados e implementa estruturas de dados organizadas em relações.

Esse modelo de estruturação de dados também restringe a repetição e perda de informação.

Técnicas de Programação - Segundo Bimestre

"Em todo nosso projeto temos várias situações, vários casos de uso que em si não aparentam ser de uma certa complexidade, porém para a utilização de cada um deles existe sempre mais algum caso de uso que precisa ser utilizado, não seria problema se não envolvesse tantos casos de uso, porém o que temos nessa situação, em todo o código, são situações onde vários casos de usos e fluxos alternativos são ligados.

Basicamente nosso projeto se consiste em um site, em um servidor em que temos cadastrados vários restaurantes, onde vários usuários podem fazer pedidos para delivery, isso tudo ao mesmo tempo. Então estamos falando de um servidor que deve sim conter uma consistência e uma suportabilidade suficientemente bastante para problemas com estresse e utilização alta.

Em todo os nossos casos de uso específicos temos pelo menos um caso de uso associado, que por sua vez é requerimento do sistema para o acesso desse outro caso de uso.

Um de nossos casos mais complexos e de difícil elaboração por conta de sua dependência de outros casos de uso é o caso de uso onde o usuário quer realizar um pedido. A elaboração de todo o código e em especial deste caso de uso pode ser bem complexa de ser realizada.

No momento em que o usuário está fazendo o seu pedido o sistema deve ter acesso a suas informações pessoais de cadastro físico para uma validação no momento do pedido, o sistema também valida as informações fornecidas referentes ao endereço e o prato selecionado anteriormente pelo usuário.

Devido a estes fatos a formação deste código deve ser muito bem estruturada para que não haja uma deficiência no desempenho do site para os usuário ou para os fornecedores. Sempre devemos pensar muito bem nos laços e nas condições, fazer um teste de desempenho profundo para verificar casos extremos de usabilidade."
